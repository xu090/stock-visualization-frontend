# 操作日志功能使用说明

## 功能概述

操作日志功能用于记录和跟踪系统中所有重要的用户操作，包括登录、创建/更新/删除概念、策略管理等活动。这有助于管理员监控系统使用情况、排查问题和进行安全审计。

## 主要功能

### 1. 自动记录操作

系统会自动记录以下操作：
- **用户认证**: 登录、登出、注册
- **概念管理**: 创建、更新、删除概念
- **策略管理**: 创建、更新、删除策略
- **管理员操作**: 管理员的概念和策略管理操作

### 2. 日志信息

每条操作日志包含以下信息：
- **时间**: 操作发生的时间
- **用户**: 操作用户的用户名和角色
- **操作类型**: 创建、更新、删除、登录、登出等
- **操作对象**: 概念、策略、用户等
- **操作详情**: 具体的操作描述
- **IP地址**: 操作来源的IP地址
- **User-Agent**: 客户端信息
- **结果**: 成功或失败
- **错误信息**: 如果失败，记录错误原因

### 3. 管理界面

在管理后台的"操作日志"标签页中，管理员可以：
- 查看所有操作日志
- 按时间排序
- 查看统计信息（总操作数、今日操作、成功/失败统计）
- 刷新日志数据

## API端点

### 获取操作日志
```
GET /api/admin/operation-logs
```

**参数**:
- `user_id` (可选): 按用户ID筛选
- `action` (可选): 按操作类型筛选
- `resource` (可选): 按资源类型筛选
- `limit` (可选): 返回数量限制，默认100，最大500
- `offset` (可选): 偏移量

**响应示例**:
```json
{
  "data": [
    {
      "id": 1,
      "userId": 2,
      "username": "admin",
      "role": "admin",
      "action": "login",
      "resource": "user",
      "resourceId": "2",
      "details": "用户登录: admin",
      "ip": "127.0.0.1",
      "userAgent": "Mozilla/5.0...",
      "result": "success",
      "errorMessage": null,
      "timestamp": 1777286307112
    }
  ]
}
```

### 获取统计信息
```
GET /api/admin/operation-logs/stats
```

**响应示例**:
```json
{
  "data": {
    "total": 7,
    "byAction": {
      "login": 2,
      "create": 3,
      "update": 2
    },
    "byResource": {
      "user": 2,
      "concept": 3,
      "strategy": 2
    },
    "byResult": {
      "success": 6,
      "failed": 1
    },
    "todayCount": 7
  }
}
```

## 数据库结构

操作日志存储在 `operation_logs` 表中：

```sql
CREATE TABLE operation_logs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT,
    username VARCHAR(64),
    role VARCHAR(16),
    action VARCHAR(32) NOT NULL,
    resource VARCHAR(64),
    resource_id VARCHAR(128),
    details TEXT,
    ip VARCHAR(45),
    user_agent TEXT,
    result VARCHAR(16) DEFAULT 'success',
    error_message TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## 操作类型

- `register`: 用户注册
- `login`: 用户登录
- `logout`: 用户登出
- `create`: 创建操作
- `update`: 更新操作
- `delete`: 删除操作
- `bootstrap`: 系统初始化操作

## 资源类型

- `user`: 用户相关操作
- `concept`: 概念相关操作
- `admin_concept`: 管理员概念操作
- `strategy`: 策略相关操作
- `admin_strategy`: 管理员策略操作
- `system`: 系统相关操作

## 使用建议

1. **定期检查**: 建议管理员定期查看操作日志，特别是失败的操作
2. **异常监控**: 关注失败的登录尝试，可能表示安全问题
3. **性能优化**: 操作日志会不断增长，建议定期清理旧数据
4. **权限控制**: 只有管理员可以查看操作日志

## 技术实现

### 后端实现
- **服务文件**: `backend/app/operation_log_service.py`
- **API集成**: 在 `backend/app/main.py` 中的各个API端点中集成日志记录
- **表创建**: 服务启动时自动创建操作日志表和索引

### 前端实现
- **组件文件**: `frontend/src/components/admin/AdminOperationLogsPanel.vue`
- **管理页面**: 在 `frontend/src/views/AdminView.vue` 中集成

## 注意事项

1. 操作日志记录可能影响系统性能，建议在高并发场景下进行性能测试
2. 敏感信息（如密码）不会记录到操作日志中
3. IP地址和User-Agent信息可能被伪造，仅作参考
4. 定期清理操作日志以避免数据库过大
